git status

新規追加されたファイルや変更が加えられたファイルを表示する。
よろしくね！

git init [--bare] [directory]

git init
カレントディレクトリをgitの作業用リポジトリにする。
具体的には、.gitディレクトリを作る。
[directory]を指定すると、指定したディレクトリに.gitディレクトリを作り、
gitリポジトリ化する。
git init --bare
カレントディレクトリをgitのリモート用(共用)リポジトリにする。
git init コマンドで作られる.gitディレクトリの中身がカレントディレクトリ内に展開される。
git init でできる作業用リポジトリとの違いは、pushされたときにファイルが即最新のコミットの状態になること。

git add [filepattern]

git add
このコマンドは新しいファイルや修正したファイルのコンテンツを索引に追加します。
結果としてそれらのコンテンツを stage する (次回のコミットに含めるようgitに指示する)ことになります。
git add "filepattern"…
コンテンツを追加するファイルを指定します。
ファイルグロブ(例 *.c)でマッチするファイルを すべて追加することができます。
また、 ディレクトリ名を指定してそのディレクトリ配下を 再帰的に全て追加することが可能です。
(例: dir/file1 と dir/file2 を追加するために dir を指定する)

git log

git log
コミット履歴を表示する。履歴のページングにはj/k(上/下)で行う
git log --oneline
コミット履歴を1行で表示する。
git log --graph
コミット履歴のブランチ間の関係をグラフ表示する。

git branch

■git で branch の作り方

      $ git branch ブランチ名
作成後は
      $ git branch
で確認できます。

■作成したブランチをリモートレポジトリに追加する方法

      $ git push

■ブランチの切り替え

      $ git branch -b ローカルブランチ名 リモートブランチ名

切り替わっているか確認したかったら

      $ git branch
をして
      * ローカルブランチ名
      master
とかなっていればOK
